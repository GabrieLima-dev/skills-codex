# Checklist de Prontidão do Projeto

## Identidade e Escopo

1. Nome do projeto.
2. Objetivo funcional principal.
3. Escopo da solicitação atual.

## Estrutura Técnica

1. Repositórios/pastas afetadas.
2. Stack backend (framework, versão) - obrigatório.
3. Stack frontend (framework, versão) - obrigatório.
4. Stack docs (ferramenta/site/docs engine, versão) - obrigatório.
5. Banco de dados e tecnologia de cache/filas (se houver).
6. Infra/deploy alvo.

## Ambiente e Operação

1. Variáveis de ambiente obrigatórias (somente nomes, nunca valores sensíveis).
2. Comando de instalação de dependências.
3. Comando de build.
4. Comando de lint.
5. Comando de testes.
6. Comando de execução local.

## Qualidade e Publicação

1. Critérios de aceite da mudança.
2. Estratégia de validação (automática e manual).
3. Estratégia de rollback.
4. Restrições de segurança/compliance.
